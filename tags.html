Tags

blargbot uses a comprehensive system of tags. The syntax for them is simple: {tagname;arguments...}. Arguments are separated with semicolons (;) and there can be any number of arguments. Some tags require more arguments than others. There are some examples here.

There are two types of tags:

  1. Simple Tags
  2. Complex Tags

Simple Tags

These tags are just simple replaces, and don't take any arguments.

Tag Name Function Example Tag Example Output

channelname Returns the name of the current channel This channel's name is #{channelname} This channel's name is #test-channel
channelpos Returns the position of the current channel This channel is in position {channelpos} This channel is in position 1
guildid Returns the id of the current guild This guild's id is {guildid} This guild's id is 1234567890123456
guildname Returns the name of the current guild This guild's name is {guildname} This guild's name is TestGuild
guildmembers Returns the number of members on the current guild This guild has {guildmembers} members. This guild has 123 members.
guildownerid Returns the id of the guild's owner The owner's id is {guildownerid} The owner's id is 1234567890123456
guildownername Returns the name of the guild's owner The owner's name is {guildownername} The owner's name is stupid cat
guildownernick Returns the nick of the guild's owner. If it doesn't exist, returns the name The owner's nick is {guildownernick} The owner's nick is cat
guildicon Returns the icon of the current guild The guild's icon is {guildicon} The guild's icon is (icon url)
guilddefaultchannelid Returns the guild's default channel's id Default channel is {guilddefaultchannelid} Default channel is 1234567890123455
guilddefaultchannelname Returns the guild's default channel's name Default channel is {guilddefaultchannelname} Default channel is defaultchannel
randuser Returns the id of a random user on the current guild. {username;{randuser}} is a lovely person! {username;{randuser}} isn't as good. abalabahaha is a lovely person! stupid cat isn't as good.

rb Will be replaced by } on execution. &rb; works too. This is a bracket! {rb} This is a bracket! }

Complex Tags

These tags are more powerful.
<> - denotes required arguments
() - denotes optional arguments
... - denotes one or more arguments

Tag Name Arguments Function Example Tag Example Output
username (name) (quiet) Returns the user's name. If name is specified, gets that user instead. If quiet is specified, if a user can't be found it will simply return the name Your username is {username;stupid;this can be anything} Your username is stupid cat
userid (name) (quiet) Returns the user's ID. If name is specified, gets that user instead. If quiet is specified, if a user can't be found it will simply return the name Your id is {userid} Your id is 123456789123456
userdiscrim (name) (quiet) Returns the user's discriminator. If name is specified, gets that user instead. If quiet is specified, if a user can't be found it will simply return the name Your discrim is {userdiscrim} Your discrim is 1234
usernick (name) (quiet) Returns the user's nickname. If it doesn't exist, returns their username instead. If name is specified, gets that user instead. If quiet is specified, if a user can't be found it will simply return the name Your nick is {usernick} Your nick is cat
useravatar (name) (quiet) Returns the user's avatar. If name is specified, gets that user instead. If quiet is specified, if a user can't be found it will simply return the name Your avatar is {useravatar} Your avatar is (avatar url)
userreply (name) (quiet) Mentions the user. If name is specified, gets that user instead. If quiet is specified, if a user can't be found it will simply return the name Hello, {userreply}! Hello, @stupid cat!
usergame (name) (quiet) Returns the game the user is playing. If name is specified, gets that user instead. You're playing {usergame} right now! If quiet is specified, if a user can't be found it will simply return the name You're playing documentation right now!
usergametype (name) (quiet) Returns how the user is playing the game (playing, streaming). If name is specified, gets that user instead. If quiet is specified, if a user can't be found it will simply return the name You're {usergametype} right now! You're playing right now!
usercreatedat (format) (name) (quiet) Returns the date the user was created. If a format code is specified, the date is formatted accordingly. Leave blank for default formatting. See the moment documentation for more information. If name is specified, gets that user instead. If quiet is specified, if a user can't be found it will simply return the name Your account was created on {usercreatedat;YYYY/MM/DD HH:mm:ss} Your account was created on 2016/01/01 01:00:00.
userjoinedat (format) (name) (quiet) Returns the date the user joined the current guild. If a format code is specified, the date is formatted accordingly. Leave blank for default formatting. See the moment documentation for more information. If name is specified, gets that user instead. If quiet is specified, if a user can't be found it will simply return the name Your account joined this guild on {userjoinedat;YYYY/MM/DD HH:mm:ss} Your account joined this guild on 2016/01/01 01:00:00
guildcreatedat (format) Returns the guild joined the current guild. If a format code is specified, the date is formatted accordingly. Leave blank for default formatting. See the moment documentation for more information. This guild was created on {guildcreatedat;YYYY/MM/DD HH:mm:ss} This guild was created on 2016/01/01 01:00:00
fallback <message> Should any tag fail to parse, it will be replaced with the fallback message instead of an error {fallback;This tag failed} {randint} This tag failed
args (index) (range) Gets user input. Specifying an index will only get the word at that location, specifying a range will get all the words between index and range. Specify range as n to get all the words from index to the end Your second word was {args;1} Input: Hello world! Output: Your second word was world!
randint (min) <max> If only max is specified, gets a random number between max and 0. If both arguments are specified, gets a random number between them. You rolled a {randint;1;6} You rolled a 5
randchoose <choices...> Chooses a random choice I feel like eating {randchoose;cake;pie;pudding} today. I feel like eating pudding today.
choose <choice> <choices...> Chooses from the given options, where choice is the index of the option selected I feel like eating {choose;1;cake;pie;pudding} today. I feel like eating pie today.
replace (toreplace) <phrase> <replacewith> Replaces the phrase with replacewith. If toreplace is specified, the tag is replaced with the new toreplace. If not, it replaces the message. I like {replace;to eat;eat;nom} cheese. {replace;cheese;ham} I like to nom ham
regexreplace (toreplace) <regex> <replacewith> Replaces the regex phrase with replacewith. If toreplace is specified, the tag is replaced with the new toreplace. If not, it replaces the message. I like {regexreplace;to consume;/o/gi;a} cheese. {regexreplace;/e/gi;n} I likn ta cansumn chnnsn.
set <name> <contents> Stores a variable. These variables are saved between sessions. {set;testvar;This is a test var}
get <name> Returns a stored variable. {get;testvar} This is a test var
math <operator> <operands...> Returns a number based on the operator and operands. Valid operators are + - * / % 2 + 3 + 6 - 2 = {math;-;{math;+;2;3;6};2} 2 + 3 + 6 - 2 = 9
if <evaluator> <arg1> <arg2> <then> <else> Evaluates arg1 and arg2 using the evaluator. If it returns true, the tag returns then. Otherwise, it returns else. Valid evaluators are == != < <= > >= {if;<=;5;10;5 is less than or equal to 10;5 is greater than 10} 5 is less than or equal to 10
lower <string> Returns string as lowercase {lower;THIS WILL BECOME LOWERCASE} this will become lowercase
upper <string> Returns string as uppercase {upper;this will become uppercase} THIS WILL BECOME UPPERCASE
user <query> (quiet) Searches for a user with given query, and returns the user's name. If quiet is specified, if a user can't be found it will simply return the query. This user's name is {user;stupid;can be anything} On succeed: This user's name is stupid cat On fail: This user's name is stupid
